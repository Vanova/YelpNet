Abstract

The Yelp Data Challenge (link it to the data challenge website) is a globally organized data challenge where in the challenge lies in creating an intelligent system that can infer semantically coherent information regarding restaurants from multi-modal data provided by Yelp, which includes images, user reviews, rating etc. In this project, we focus primarily on image data and the goal is to be able to generate multiple attributes for a given image of the restaurant and predict semantically aligned labels.

We formulate our problem to be of multilabel image annotation nature and explore different deep learning architectures that are suitable for our purpose. We develop a unique system that can predict multiple labels from an image of a restaurant. In this project we show how features learnt by deep learning architectures can be used to annotate a restaurant image with multiple labels.

Introduction

Traditional search techniques include simple word based search, where-in the user searches using a specific keyword and the results are generated from the list of keywords provided by other users. This method doesnâ€™t give a user the flexibility to be able to search using keywords that are semantically aligned for the user and easy to interpret.

Multilabel image annotation is one of the most important challenges in computer vision with real world applications. Recently the area of deep learning using convolutional neural networks (CNNs) has made incredible strides in single image classification and object detection tasks. Single label image classification has been extensively studied by the deep learning community with the most advanced results reported by Resnet (link to the resnet paper) achieving near human accuracy. Images on the internet are generally associated with sentences or descriptions that capture the underlying essence of the image, rather than single label, and hence it is an important problem to be able to predict correct multiple labels. For example restaurant images uploaded on Yelp are accompanied with descriptions and reviews of the restaurant, along with several tags regarding the priciness of the restaurant, ambience etc. which provides a much richer representation of the visual content of the image. As a result we found it quite interesting and challenging to be able to predict multiple labels for the restaurant images that align with what humans actually perceive. Our primary goal is to develop a single system that learns a joint representation for multiple labels using specially designed loss functions, in effect capturing the relationship between multiple attributes.

In this project, we try to address these issues by developing a multi-task learning system that builds upon the state-of-the art Convolutional Neural Networks (CNN) based deep learning architectures. Major contributions include -

  • Developing a single CNN based system that learns multiple attributes using noisy data
  • Researching and implementing other multi-label loss functions in literature

Information about the Dataset

For our purposes, we focus on the image data provided by Yelp. This data includes 200,000 pictures of various restaurants and business, along with 77k+ businesses ids. Each image is associated with a photo-id which maps to a business-id. A single business can have multiple images linked to it. As our initial focus is to multiple attribute annotation, we mainly focus on the attributes of each business. These attributes were generated by Yelp for each business from user reviews. A sample business id is shown below -

  • "business_id": "5UmKMjUEUNdYWqANhGckJw"
  • "attributes": {"Take-out": true, "Drive-Thru": false, "Good For": {"dessert": false, "latenight": false, "lunch": false, "dinner": false, "brunch": false, "breakfast": false}, "Caters": false, "Noise Level": "average", "Takes Reservations": false, "Delivery": false}

Challenges

  • Noisy data

    • The image dataset provided is quite noisy. There are images in the dataset that include photographs of outdoor scenes, leisure photos etc which are not at all related to a restaurant.

  • Attributes are skewed

    • Distribution of attributes amongst images is not uniform. There are some attributes that occur more frequently than others. For example, Accepts Credit Card is the most commonly tagged attribute. Also as attributes were taken from yelp curators/user reviews, all businesses do not have all attributes which poses another challenge for us.

Our Methodology

Our system was developed using Caffe framework, a very popular open source platform for deep learning development. We converted all of our training and test data to the LMDB format in order to facilitate faster development. Rather than training our network from scratch for our data, we resort to fine-tuning where in the weights learnt from a previously trained network are used to initialize the network, resulting in faster convergence. In Caffe, this is possible by renaming the last fully connected layers (fc-layers) and loss layers. Last fully connected layer contains the most visual information of an image and as a result only the last layer is trained. We start with fine-tuning AlexNet, a very popular CNN architecture developed by Alex Krizhevsky which was trained on 1.2 million images for 360,000 iterations.

Our network can be visualized here.

Our next step was to fine-tune on a much deeper network. For this purpose we fine-tune our data on GoogLeNet, a much deeper network which was trained for 2400000 iterations for 60 epochs over the ImageNet data.

This architecture can be visualized here.

For single label classification, Softmax loss is used which predicts a single class out of multiple classes based on its probability. As our purpose entails multi-label task, Softmax loss cannot be used for our purpose. To learn a joint representation among multiple attributes, different loss functions need to be derived for this purpose.

One loss function that is used by us is the Sigmoid Cross Entropy Loss. Sigmoid Cross Entropy loss calculates the loss amongst various classes by trying to find the measure of similarity between true distribution and the predicted distribution of labels.
